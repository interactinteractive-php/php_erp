SQL ALIASES

    
    Энэ нь өгөгдлийн сангийн хүснэгт эсвэл хүснэгтийн тодорхой багануудад түр зуурын байдлаар нэр оноох SQL хэлний арга хэрэгсэл юм. Түр оноох учираас query ажиллаж байх хугацаанд тухайн нэршил нь хүчинтэй байдаг.

Хэрэглээ: Alias -г бид ихэвчлэн SQL statement -н бичиглэл багасгах, давхар нэршилээс зайлс хийх мөн баганы нэрийг зөв утгатайгаар буюу хүнд ойлгомжтой байдлаар харуулах зорилгоор ашигладаг. Энэ нь цаг хугацаа хэмнэх үр ашигтай бөгөөд ойлгомжтой байдлыг нэмэгдүүлдэг. Мөн олон баганыг нэг дор бичиж, нэгдсэн нэг нэршил онооход хэрэглэж болно.

Бичигдэх хэлбэр /syntax/:
Хүснэгтийн баганы нэрийг alias хийхдээ дараах байдлаар бичнэ.
SELECT column_name AS alias_name FROM table_name;

Хүснэгтийг нэрийг alias хийхдээ дараах байдлаар бичнэ.
SELECT column_name(s) FROM table_name AS alias_name;

Жишээ-1: Customers хүснэгтийн CustomerId болон CustomerName багануудын нэрийг хүнд харагдах хэлбэрийн хувьд Alias хийж үзье. 

SELECT CustomerID AS ID, CustomerName AS Customer FROM Customers;

Зураг 01 -с дээрх SELECT query хэрхэн ажилласан байгааг харна уу.

    
   
                                                                                                                                                  Зураг 01 Баганы нэрийг alias хийсэн хэрэглэгчдийн id болон нэрийн жагсаалт		    
   
Жишээ-2: Хүснэгтийн нэрийг Alias хийх гол хэрэгцээ шаардлага нь тухайн хүснэгтийн нэр хэт урт байх эсвэл олон газар бичигдэх байдаг. Энэ үед Alias ашиглан товч нэр өгсөнөөр бичиглэл багасаж, ойлгомжтой байдлыг нэмэгдүүлдэг. Дараах Select query -с хүснэгтийн нэрийг хэрхэн Alias хийж болохыг харцгаая.

SELECT o.OrderID, o.OrderDate, c.CustomerName FROM Customers AS c, Orders AS o WHERE c.CustomerName = "Around the Horn" AND c.CustomerID = o.CustomerID;

Зураг 02 -с дээрх SELECT query хэрхэн ажилласан байгааг харна уу.
    
   
Зураг 02 Customers болон Orders хүснэгтүүдийн нэрийг alias хийж давхар select хийж байна.		    
   
Дээрх жишээнд олон хүснэгтийг нэгтгэж, харуулахдаа хүснэгтүүдийн нэрийг товч нэршилээр оноож харуулсан байна. Олон хүснэгт нэг дор select хийх нь SQL JOIN хийхээс өөр төрлийн ойлголт бөгөөд аргачлал нь шүүлт хийх замаар үр дүнгээ гаргаж харуулдаг. Тухайлбал Customers хүснэгтийн CustomerName нь 'Around the Horn' утгатай мөн CustomerID нь Orders хүснэгтийн CustomerID -тай тэнцэж байх үеийн бичлэгүүдийг OrderID, OrderDate, CustomerName зэрэг талбаруудаар харуулсан байна. Энэ үед нэг жагсаалтанд өөр хоорондоо холбоотой хүснэгтүүдийг холбож байгаа учир шаардлагатай багануудын нэрийг alias_name.column_name хэлбэртэй бичиж дууддаг.  

Жишээ-3: Бид хаяг байршилын мэдээллийг олон жижиг хэсэгт хуваан өгөгдлийн санд хадгалдаг. Харин нэгтгэсэн байдлаар харуулах шаардлага тулгардаг. Энэ тохиолдолд олон багана зэрэгцүүлж, нэг нэршил оноож болохыг дараах select query -с харцгаая.

SELECT CustomerName, Address + ', ' + PostalCode + ' ' + City + ', ' + Country AS Address FROM Customers;
