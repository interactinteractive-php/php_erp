SQL UNIQUE Constraint

    
    Энэ төрлийн хязгаарлалт нь хүснэгтийн тухайн баганы утгууд өөр хоорондоо ялгаатай байх ёстойг илтгэдэг. Өөрөөр хэлбэл тухайн баганад нэг утга нэг л оршино гэсэн үг юм.  

Хэрэглээ: Unique болон Primary Key хязгаарлалтууд нь үндсэндээ ижил зарчимтай боловч хоорондоо ялгаатай юм. Primary Key хязгаарлалт нь анхнаасаа unique байхаар тодорхойлогдсон бөгөөд тухайн хүснэгтэд зөвхөн нэг л Primary Key байх ёстой. Харин Unique хязгаарлалтыг шаадлагатай гэж үзсэн баганууддаа тодорхойлж өгөх боломжтой.

Жишээ-1: Persons хүснэгтийг үүсгэхдээ ID багана нь Unique байх ёстой гэдгийг дараах байдлаар тодорхойлж бичнэ. 

SQL Server / Oracle / MS Access:
CREATE TABLE Persons (
    ID int NOT NULL UNIQUE,
    LastName varchar(255) NOT NULL,
    FirstName varchar(255),
    Age int
);

MySQL:
CREATE TABLE Persons (
    ID int NOT NULL,
    LastName varchar(255) NOT NULL,
    FirstName varchar(255),
    Age int,
    UNIQUE(ID)
);

Жишээ-2: Persons хүснэгтийг үүсгэхдээ ID болон LastName багануудыг Unique байх ёстой гэдгийг дараах байдлаар тодорхойлж бичнэ. 

MySQL / SQL Server / Oracle / MS Access:
CREATE TABLE Persons (
    ID int NOT NULL UNIQUE,
    LastName varchar(255) NOT NULL,
    FirstName varchar(255),
    Age int,
    CONSTRAINT UC_Person UNIQUE (ID, LastName)
);

Дээрх query -д CONSTRAINT командаар UC_Person нэртэй UNIQUE хязгаарлалтыг ID болон LastName баганууд дээр үүсгэхийг тодорхойлсон байна.

Жишээ-3: Тухайн хүснэгтийг аль хэдийн үүсгэсэн бол ALTER TABLE командаар засахдаа Constraint буюу хязгаарлалт тодорхойлж болно. Persons хүснэгтийг үүсгэсэн гэж үзээд ID талбарыг Unique байх ёстой гэдгийг дараах байдлаар тодорхойлж бичнэ. 

MySQL / SQL Server / Oracle / MS Access:
ALTER TABLE Persons ADD UNIQUE (ID);

Constraint команд ашиглан 1 -с олон баганаар хязгаарлалт үүсгэх бол дараах байдлаар бичиж болно.
ALTER TABLE Persons ADD CONSTRAINT UC_Person UNIQUE (ID, LastName);

Жишээ-4: Тухайн үүсгэсэн хязгаарлалт шаардлагагүй гэж үзвэл ALTER TABLE DROP командын тусламжтай устгах бол дараах байдлаар бичнэ.

MySQL:
ALTER TABLE Persons DROP INDEX UC_Person;

SQL Server / Oracle / MS Access:
ALTER TABLE Persons DROP CONSTRAINT UC_Person;
