SQL SELF JOIN

    
    Энэ нь JOIN хийх төрлүүдийн нэг бөгөөд тухайн холбогдсон хүснэгтүүдийн бүхии л бичлэгүүдийг харуулах зорилготой SQL хэлний түлхүүр үг юм. Гол зарчим нь баруун зүүн гэж ялгалгүй бүх бичлэгүүдийг select хийх юм.

Хэрэглээ: Өгөгдлийн санд тийм ч их хэрэглэгдээд байдаггүй ч 2 болон түүнээс дээш хүснэгтүүдийн бүх бичлэгүүдийг харах, өгөгдлийн давхардал илрүүлэх, анализ хийж дараа нь ажиллуулах ямар төрлийн query бичих ээ шийдвэрлэх гэх зэрэг олон чухал зүйлсэд хэрэглэх боломжтой. 




Бичигдэх хэлбэр /syntax/:
SELECT column_name(s) FROM table1 FULL OUTER JOIN table2 ON table1.column_name = table2.column_name;

Зураг 01 -с жишээнд ашиглагдах Customers хүснэгтийг харна уу.

    
                                                                                                                                                                                                     Зураг 01 Customers хүснэгт    
    Зураг 02 -с жишээнд ашиглагдах Orders хүснэгтийг харна уу.    
   
Зураг 02 Orders хүснэгт		    
   
Жишээ-1: Дээрх 2 хүснэгтийг Full Join хийж ямар үр дүн гарахыг харцгаая. Гол зарчим нь тухайн 2 хүснэгтийн бүх бичлэгүүдийг харуулах юм. Хэрэв өөр хоорондоо тохирохгүй бичлэгүүд байвал мөн л харуулах юм.

SELECT Customers.CustomerName, Orders.OrderID FROM Orders FULL OUTER JOIN Customers ON Customers.CustomerID=Orders.CustomerID ORDER BY ORDER BY Customers.CustomerName;

Зураг 03 -с дээрх SELECT query хэрхэн ажилласан байгааг харна уу.

    
                                                                                                                        Зураг 03 Customer болон Orders хүснэгтүүдийг FULL JOIN хийж байна.    
   
Зурагт FULL JOIN -ны хэрэглээг харуулж байгаа бөгөөд Customers хүснэгтийн CustomerName, Orders хүснэгтийн OrderID зэрэг талбаруудаар select хийж харгалзах үр дүнг CustomerName талбараар өсөхөөр эрэмбэлж, харуулсан байна.
