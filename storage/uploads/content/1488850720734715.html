SQL HOSTING

    
    Энэ нь Sql илэрхийллийн тусламжтайгаар тодорхойлогдсон хийсвэр хүснэгтийн харагдац юм. View нь жинхэнэ хүснэгтийн бүтэцтэй адилхан. 

Хэрэглээ: Томоохон өгөгдлийн сантай ажиллах үед Sql query нөсөр том урт бичиглэлтэй болдог. Sql командуудыг хэрэгжүүлэхэд View ашиглаж болно. Энэ нь бичиглэл багасгах юм. Мөн Sql View нь үргэлж өгөгдлийн сангийн хүснэгтээс бичлэгүүдээ шинэчилсэн хэлбэрээр татаж харуулдаг давуу талтай.

Бичигдэх хэлбэр /syntax/:

View үүсгэх
CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

View засах 
CREATE OR REPLACE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

View устгах
DROP VIEW view_name;

Жишээ-1: Дараах query нь Products хүснэгтээс зарах боломжгүй бүтээгдэхүүнийг шүүж View үүсгэж байна.

CREATE VIEW [Current Product List] AS
SELECT ProductID, ProductName
FROM Products
WHERE Discontinued = No;

Үүсгэсэн View -г дуудаж харахдаа дараах байдлаар бичнэ.

SELECT * FROM [Current Product List];

Жишээ-2: Дараах query нь 1997 онд худалдаалагдсан бүтээгдэхүүнийг харуулах View -с ангилалаар нь бүлэглэж тухайн ангилалын бүтээгдэхүүнүүд нийт хичнээн төгрөгийн орлого оруулсан болох харуулах View үүсгэж байна.

CREATE VIEW [Category Sales For 1997] AS
SELECT DISTINCT CategoryName, Sum(ProductSales) AS CategorySales
FROM [Product Sales for 1997]
GROUP BY CategoryName;

Үүсгэсэн View -г дуудаж харахдаа дараах байдлаар бичнэ.

SELECT * FROM [Category Sales For 1997];

Мөн үүсгэсэн View -д шүүлт тавьж харах боломжтой. Дараах query нь 1997 онд борлуулагдсан уух зүйл ангилалын бүтээгдэхүүнүүдийн нийт мөнгөн дүн болон ангиллын нэрийг харуулна. 

SELECT * FROM [Category Sales For 1997]
WHERE CategoryName = 'Beverages';
