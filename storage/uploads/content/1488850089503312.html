SQL RIGHT JOIN

    
    Энэ нь JOIN хийх төрлүүдийн нэг бөгөөд SQL хэлний түлхүүр үг юм. Гол зарчим нь баруун талд байрлах (table2) хүснэгтийн бүх бичлэгүүдийг зүүн талын хүснэгтийн тохирох бичлэгүүдийн хамт харуулдаг.  Хэрэв тохирохгүй бичлэг таарвал зүгээр л null буюу хоосон утгатай харуулдаг. Өөрөөр хэлбэл Left Join -ны эсрэг чиглэлд ажилладаг юм.

Хэрэглээ: Right Join -г холбоотой хүснэгтүүдээс аль нэгийх нь бүх талбарыг шаардлагатай талбаруудын хамт нэг илэрцээр харах шаардлага гарсан үед ашиглана. Өөрөөр хэлбэл зөвхөн баруун талд буй хүснэгтийг чухалчлан авч үзэх бөгөөд зүүн талд буй хүснэгтийг тооцоолол хийхдээ ашиглаж шаардлагатай талбаруудыг  авах замаар олон төрлийн үр дүнг гаргах боломжтой. 



Бичигдэх хэлбэр /syntax/:
SELECT column_name(s) FROM table1 RIGHT JOIN table2 ON table1.column_name = table2.column_name;

Зураг 01 -с жишээнд ашиглагдах Employees хүснэгтийг харна уу.

    
   
                                                                                                                                    Зураг 01 Employees хүснэгт		    
    Зураг 02 -с жишээнд ашиглагдах Orders хүснэгтийг харна уу.    
   
Зураг 02 Orders хүснэгт		    
   
Жишээ-1: Нэг ч захиалга бүртгэж аваагүй ажилтаныг нэрийг харах шаардлагатай болсон гэж үзье. Энд нэг ажилтан нь олон захиалга бүртгэж болох бөгөөд Right Join зарчим ёсоор Employees хүснэгтийн бүх талбарыг Orders хүснэгтийн тохирох талбаруудын хамт харагдах ёстой. Хэрэв Orders хүснэгтээс тухайн ажилтанд тохирох утга олдохгүй бол null утгатай харагдах бөгөөд захиалга бүртгээгүй ажилтан болох юм. 

SELECT Orders.OrderID, Employees.LastName, Employees.FirstName FROM Orders RIGHT JOIN Employees ON Orders.EmployeeID = Employees.EmployeeID ORDER BY Orders.OrderID;

Зураг 03 -с дээрх SELECT query хэрхэн ажилласан байгааг харна уу.

    
   
Зураг 03 Employees болон Orders хүснэгтүүдийг RIGHT JOIN хийж байна.		    
   
Зурагт Orders хүснэгтийн EmployeeID талбарын утга Employees хүснэгтийн EmployeeID -тай тэнцүү байх үеийн Employees хүснэгтийн LastName, FirstName багануудын бүх бичлэгүүдийг Orders хүснэгтийн OrderID баганы тохирох бичлэгүүдийн хамт select хийж харуулсан байна. NULL эсвэл Хоосон утгатай харагдаж байгаа талбарууд бол зүүн талын хүснэгтэд тохирох утга байгаагүйн илрэл буюу манай тохиолдолд захиалга бүртгээгүй ажилтаныг илэрхийлсэн байна.
