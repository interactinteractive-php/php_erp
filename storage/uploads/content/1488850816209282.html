SQL ISNULL, NVL, IFNULL, COALESKE

    
    Эдгээр нь хоосон буюу ямар ч утгагүй талбарын утгыг математик тооцоололд ашиглахаас сэргийлж шалгалт хийх SQL хэлний функцууд юм.

Хэрэглээ: Тоон төрлийн утга агуулах талбарыг сонголтын буюу Null утга авахаар тохируулах тохиолдол гардаг. Харин тооцоолол хийж байхад Null утгатай талбар таарахад тооцооллын үр дүн Null болно.
   
Зураг 01 -с жишээнд ашиглах хүснэгтийг харна уу.

    
   
                                                                                                                                                Зураг 01 Null шалгах жишээнд ашиглах Products хүснэгт		    
   
Зурагт Products хүснэгтийн UnitsOnOrder багана нь сонголтын буюу хоосон утгатай байж болохоор байгаа нь харагдаж байна.

Жишээ-1: Дээрх Products хүснэгтийн UnitsOnOrder баганыг дараах тооцоололд ашиглах шаардлага гарсан гэж үзье. Энд тухайн бүтээгдэхүү дэлгүүрт байгаа тоо болон захиалагдсан тоо хэмжээний нийлбэрийг нэгж үнээр үржүүлж тухайн бүтээгдэхүүний нийт мөнгөн дүнг харуулах гэж байна.

SELECT ProductName, UnitPrice*(UnitsInStock+UnitsOnOrder) FROM Products;

Дээрх query -д UnitsInOrder баганыг Null утгатай байгаа эсэхийг шалгаагүй байгаа учир үр дүнд зарим талбарын утга NULL болж гарна. Хэрэв хоосон утгатай байгааг шалгаж тооцоололд ашиглавал дээрх асуудал шийдэгдэнэ. Энэ тохиолдолд бид Null утгын оронд 0 -г орлуулж тооцоолол хийнэ.
 
Өгөгдлийн сан удирдах системүүдэд өөр өөр нэртэй функцуудыг ашигдадаг.


MS Access->ISNULL()
SELECT ProductName, UnitPrice*(UnitsInStock+IIF(ISNULL(UnitsOnOrder), 0, UnitsOnOrder)) FROM Products;

SQL Server->ISNULL()
SELECT ProductName, UnitPrice*(UnitsInStock+ISNULL(UnitsOnOrder, 0)) FROM Products;

Oracle->NVL()

SELECT ProductName, UnitPrice*(UnitsInStock+NVL(UnitsOnOrder, 0)) FROM Products;


MySQL->IFNULL(), COALESKE() 

IFNULL() ашиглах бол 
SELECT ProductName, UnitPrice*(UnitsInStock+IFNULL(UnitsOnOrder, 0)) FROM Products;

COALESCE() ашиглах бол
SELECT ProductName, UnitPrice*(UnitsInStock+COALESCE(UnitsOnOrder, 0)) FROM Products;
