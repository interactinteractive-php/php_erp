SQL INNER JOIN

    
    Энэ нь өгөгдлийн сангийн 2 болон түүнээс дээш хүснэгтүүдээс харгалзах түлхүүр утгаараа хоорондоо ижил байх үеийн бүх бичлэгүүдийг select хийж харах боломж олгодог SQL хэлний түлхүүр үг юм.

Хэрэглээ: Inner Join -г хоорондоо 1-1, 1-*, *-1 холболтын төрлүүдээр холбогдсон хүснэгтүүдээс шаардлагатай талбаруудыг нэг илэрцээр харах шаардлага гарсан үед ашиглана.

Бичигдэх хэлбэр /syntax/:
SELECT column_name(s) FROM table1 INNER JOIN table2 ON table1.column_name = table2.column_name;

Зураг 01 -с жишээнд ашиглагдах Customers хүснэгтийг харна уу.

    
   
                                                                                                                                    Зураг 01 Customers хүснэгт		    
    Зураг 02 -с жишээнд ашиглагдах Orders хүснэгтийг харна уу.    
   
Зураг 02 Orders хүснэгт		    
   
Жишээ-1: Дээрх Orders хүснэгтэд чухам хэн гэдэг хүн захиалга хийсэн гэдэг нь тодорхой бус зөвхөн тухайн хэрэглэгчийн давтагдашгүй дугаарыг л харуулсан байна. Эндээс нэг хэрэглэгч нь олон захиалга хийж болох учир Orders хүснэгтэд Customers хүснэгтийн PK буюу анхдагч түлхүүрийг FK буюу гадаад түлхүүр болгон тодорхойлж нэг олон холбоос үүсгэсэн байна. Харин захиалгын ажилтан тухайн захиалгыг хэн гэдэг нэртэй хэрэглэгч хийсэн эсэхийг шалгаж, мэдэхийн тулд Inner Join ашиглана. 

SELECT Orders.OrderID, Customers.CustomerName FROM Orders INNER JOIN Customers ON Orders.CustomerID = Customers.CustomerID;

Зураг 03 -с дээрх SELECT query хэрхэн ажилласан байгааг харна уу.

    
   
Зураг 03 Orders болон Customers хүснэгтүүдийг JOIN хийж байна.		    
   
Зурагт Orders хүснэгтийн CustomerID талбарын утга Customers хүснэгтийн CustomerID талбарын утгатай тэнцүү байх үеийн бүх бичлэгүүдийг select хийж харуулсан байна. Orders хүснэгтэд 5 бичлэг байсан бол JOIN хийсний дараа 4 бичлэгтэй харагдаж байна. Энэ нь хүснэгтүүд зөвхөн харгалзах түлхүүрээрээ тэнцэх үед л select хийж харуулна гэсэн санаа юм. Өөрөөр хэлбэл Inner Join -ны ажиллах зарчим.
